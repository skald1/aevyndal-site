# ÆVYNDAL INTERACTION SYSTEM

## Purpose

This document defines how users interact with aevyndal.com.

Interactions should feel natural.

Nothing should feel mechanical.

Nothing should remind users they are using software.

Interactions should create trust.

---

# Philosophy

The interface is human.

The world is ancient.

Interactions should feel like invitations.

Not commands.

Curiosity should pull people forward.

Nothing should push.

---

# Interaction Hierarchy

World

↓

Atmosphere

↓

Glass

↓

Information

↓

Interaction

Interactions sit above information.

They should never dominate.

---

# Hover States

Hover should reveal.

Not surprise.

Effects:

* Slight brightening
* Slight elevation
* Tiny scale increase

Scale:

1.01–1.03

Never larger.

Hover should whisper.

Never shout.

---

# Buttons

Buttons should feel soft.

Rounded.

Subtle.

Never glossy.

Never flashy.

Hover:

* brighten slightly
* border becomes clearer

No bounce.

No aggressive animation.

---

# Cards

Cards are windows.

Hover should suggest:

"There is more."

Not:

"Click me."

Effects:

* slight lift
* brighter border
* image movement
* metadata reveal

Cards should feel alive.

---

# Search

Search is king.

Search should always feel available.

Never hidden.

Never buried.

Search should invite curiosity.

Not urgency.

---

# Search Results

Grouped:

Locations

People

Creatures

Faiths

Articles

Campaigns

Results should answer:

"What might interest me?"

Not:

"What exactly matches?"

Discovery matters.

---

# Navigation

Navigation should disappear into the world.

Users should rarely think about it.

Above fold:

Floating.

Integrated.

Below fold:

Ghost header.

Subtle.

Always accessible.

---

# Links

Links should feel calm.

Underline on hover.

Subtle color changes.

Avoid bright blue hyperlinks.

Links should belong to the page.

---

# Forms

Minimal.

Simple.

Comfortable spacing.

Rounded edges.

Soft borders.

Forms should never feel corporate.

---

# Filters

Avoid excessive filters.

Discovery is preferred over filtering.

Too many filters make the world feel mechanical.

---

# Pagination

Avoid page numbers where possible.

Prefer:

Load More

Infinite relationships

Collections

Discovery

Pages should feel continuous.

---

# Related Content

Never show:

"More Articles"

Instead show:

Nearby Places

Related Creatures

People Who Live Here

Stories From This Region

Campaigns Set Here

Connections matter.

Not categories.

---

# Age Switching

Changing ages should feel like travel.

Not theme switching.

Atmosphere changes continuously.

Information changes separately.

The world remains.

---

# Scroll Behavior

Scrolling should feel like walking.

Not consuming.

Large spacing.

Natural rhythm.

No scroll-jacking.

No forced movement.

---

# Microinteractions

Subtle.

Examples:

Search focus glow.

Card image drift.

Border brightening.

Metadata fade.

Tiny icon movement.

Nothing should become entertainment.

---

# Loading

Avoid loading screens.

Skeletons only when necessary.

Prefer immediate content.

The world should remain visible.

Never blank the screen.

---

# Error States

Friendly.

Quiet.

Never alarming.

Errors should feel temporary.

Not catastrophic.

---

# Empty States

Invite curiosity.

Suggest places.

Suggest stories.

Suggest relationships.

Never display:

"No Results"

Instead:

"Nothing here yet."

or

"Try exploring somewhere nearby."

---

# Accessibility

Keyboard first.

Visible focus.

Reduced motion support.

Touch targets generous.

Interaction must remain effortless.

---

# Mobile

Thumb-first.

Comfortable spacing.

Search always reachable.

Cards stack naturally.

Nothing should feel cramped.

---

# Final Feeling

People should never think:

"This UI is impressive."

They should think:

"This feels comfortable."

And eventually:

"I know my way around here."
